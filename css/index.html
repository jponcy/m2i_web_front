Le CSS

Définition

Signifie Cascading Style Sheet.

    Style
    Car ce language permet de gérer les styles/visuels de notre application
    Sheet/feuille
    Générallement stockés dans d'autres fichiers que le HTML, ces fichiers sont nommées : stylesheets / feuilles de style
    Cascading
    Les styles s'appliquent en cascade. C'est à dire qu'un style appliqué sur une balise sera aussi répercuté (appliqué) sur les balises présentes à l'intérieur de cette dernières. On parle générallement avec les termes suivants : les styles d'une balises s'appliquent sur leurs (balises) enfants.

Où placer son code CSS

Le code CSS peut-être placé de plusieurs manières différentes :

directement dans l'attribut style d'une balise CSS
cette solution n'est pas recommandé. Elle ne fournit que peu d'avantage (en dehors de sa facilité)
Exemple : <p style="color: red;"> Bonjour tout le monde !</p>
dans une balise <style> présente dans la partie <head>
cette solution est générallement utilisé pour y placer du style spécifique pour 1 fichier donné.
Exemple : 
<!DOCTYPE html>
<html lang="en">
<head>
  [...]
  <style>
    strong {
      color: red;
    }
  </style>
</head>
[...]
</html>
dans une feuille de style
c'est la solution recommandé, cela permet un certain nombre d'avantage comme :
  • code CSS totallement séparé du HTML
  • une feuille de style peut-être utiliser par nos fichiers HTML en une seule ligne (pas de duplication du CSS)
  • possibilité de mise en cache du fichier CSS (le fichier CSS étant générallement le même pour toutes nos pages, il ne sera demandé/téléchargé qu'une seule fois)
Le fichier contenant le code CSS devra avoir l'extension CSS. Le fichier principal de style se nomme générallement style.css OU app.css. Nous l'importerons avec la ligne suivante (placée dans la partie <head>) : <link rel="stylesheet" href="style.css">

Quelques propriétés CSS

color
Choix de la couleur de police d'écriture
background-color
changement de la couleur de fond
font-family
Changement de la police d'écriture. Plusieurs noms peuvent être fournis. Les noms de polices sont lus de gauche à droite. La première trouvée/disponible est concervé.
font-size
Changement de la taille de la police (exprimé la taille : rem, em, pt, %, px)
  • rem => pourcentage par rapport à la taille de police configuré sur le body
  • em => pourcentage par rapport à la taille de la police
  • pt => taille de police
  • % => pourcentage par rapport à la taille de la balise
  • px => taille en pixel (pas fiable dû aux différences de densités de certains écrans)

margin-top
Choix de la marge extérieur au dessus. On a les variantes pour les autres directions (right, left, bottom)
padding-top
Choix de la marge intérieur au dessus. On a les variantes pour les autres directions (right, left, bottom)
margin/padding
On parle de super propriété. Elles permettent de saisir plusieurs valeurs en même temps. Par exemple : 
margin: 10px;
/* Revient à la même chose que : */
margin-top: 10px;
margin-right: 10px;
margin-bottom: 10px;
margin-left: 10px;

margin: 5px 10px 5px 10px;
/* Revient à la même chose que : */
margin: 5px 10px;
/* Revient à la même chose que : */
margin-top: 5px;
margin-right: 10px;
margin-bottom: 5px;
margin-left: 10px;
border
Super propriété permettant de saisir en un coup les propriétés suivantes :
  • border-top-color
  • border-top-style
  • border-top-width
  • border-right-color
  • border-right-style
  • border-right-width
  • border-bottom-color
  • border-bottom-style
  • border-bottom-width
  • border-left-color
  • border-left-style
  • border-left-width
border-radius
gestion des bordures arrondis
overflow-x, overflow-y, overflow
Gestion des dépassements (pour afficher ou non une scroll bar).
width/heigth
Gérer la largeur/hauteur
Exemples : 
width: 50%;
heuteur: 10vh;
